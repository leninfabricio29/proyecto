INTRODUCCIÓN

El desarrollo de software bajo políticas de desarrollo sustentable permite la construcción de proyectos informáticos ágiles, robustos, reusables y sobre todo que permitan el mejoramiento de actividades humanas. Sin embargo, la construcción de software requiere de una serie de metodologías y secuencia de actividades necesarias para asegurar la calidad del sistema, adicionalmente, la construcción del software requiere una planificación adecuada partiendo desde el establecimiento del alcance y qué objetivos se desea alcanzar. El presente informe, busca detallar la temática y los objetivos deseados durante la construcción de una plataforma web para el restaurante Carbonero Balcón ubicado en la ciudad de Loja. Además, se definen las metodologías, técnicas y arquitecturas que se implementarán como primer paso en la elaboración del sistema.

DESARROLLO

2.1. Temática.

Plataforma WEB para la automatización de pedidos en el restaurante Carbonero Balcón de la ciudad de Loja.

2.2. Objetivos

Diseñar una plataforma web para la automatización de los pedidos en el restaurante Carbonero Balcón en la ciudad de Loja bajo las normativa de desarrollo de software sustentable.

  • Analizar los procesos que se llevan a cabo en el restaurante referente a la recepción de pedidos. Conocer los conceptos relacionados a las metodologías de desarrollo de software sustentable.
  • Construir una solución informática para la automatización de pedidos en el restaurante utilizando una metodología ágil.
  • Implementar la solución informática en el tiempo establecido por el cliente.

    • 2.4. Tecnologías (lenguajes, metodologías, plataformas).

    Durante el desarrollo del proyecto se implementará una serie de herramientas para la construcción de modelos, implementación, codificación y puesta en marcha del mismo.

    Los lenguajes de programación ayudan a los desarrolladores en la construcción de sistemas robustos, donde su implementación está netamente definida por el contexto y el sector de implementación.
    Sin embargo, primeramente se debe partir de la definición precisa sobre qué es un lenguaje de programación, de tal forma que, un lenguaje de programación, según Carlos Ureña Alamagro, se define como el conjunto de reglas y normas que permiten asociar cada programa con su fin objetivo y la forma en la que se va a desplegar. Otra definición lo muestra como, un acuerdo o convenio de las reglas que se deben seguir para la construcción de programas informáticos.
    En este contexto, se puede determinar que, un lenguaje de programación se puede definir como la representación formal de un conjunto de normas e instrucciones que permiten al desarrollador escribir una serie de órdenes, acciones, operaciones y algoritmos bajo el término de programa para controlar el funcionamiento de hardware y software.
    Durante el presente proyecto se implementará una serie de lenguajes enfocados al desarrollo web, entre los que se va a implementar esta:

  • JavaScript

    • Es un lenguaje de programación enfocado a la web, el cual permite implementar funciones de primera clase al ser un lenguaje ligero, interpretado y compilado bajo la filosofía de “just-in-time”. Es implementado en entornos como Nodejs, Apache, Adobe, entre otros.
    Otra definición, según Mozilla, es un lenguaje de programación basado en prototipos, multiparadigma, de un solo hilo, dinámico, con soporte para programación orientada a objetos, imperativa y declarativa (por ejemplo programación funcional).
    En definitiva, el lenguaje permite el desarrollo ágil de aplicaciones enfocadas a la web basadas en la arquitectura de cliente-servidor tal es el caso de la plataforma del presente proyecto.

  • Python

    • Python es un lenguaje de programación de alto nivel que soporta múltiples implementaciones en el desarrollo de aplicaciones web, entre otros. En esencia, forma parte de la base del framework Django siendo el lenguaje que se implementa para el desarrollo de toda la aplicación completa.
    Otra de las definiciones que recibe python, según Python, se define como un lenguaje de programación de alto nivel orientado a objetos en compañía de una semántica agregada para el desarrollo web bajo la normativa de desarrollo rápido de aplicaciones o RAD.
    Finalmente, es importante mencionar que, el uso del lenguaje viene orientado por el uso del framework de desarrollo Django, el cual permite el desarrollo de aplicaciones web ágiles.

    Los lenguajes de base de datos permiten manejar sentencias para la manipulación de información en una base de datos relacional o no relacional, con el objetivo de mantener la información a disposición de la plataforma. Estos lenguajes se basan en sentencias de tipo SQL que permiten la gestión y recuperación de información de motores de base de datos de tipo relacional implementando el cálculo relacional y álgebra relacional.
    Entre las características del lenguaje SQL se pueden mencionar:

  • Permite crear, modificar y borrar esquemas de relación.
  • Usa el álgebra y el cálculo relacional para la realización de consultas.
  • Contiene reglas basadas en comandos para asegurar la integridad de los datos.
  • Inclusión de instrucciones de lenguajes de programación como C++, Pascal, Java, PHP, entre otros.

  • PostgreSQL

    • PostgreSQL es un sistema de base de datos para el manejo de volúmenes de información considerables provenientes de sitios web, soportando múltiples tablas índices, triggers, entre otros componentes.
    Una característica interesante de este tipo de base de datos es su no dependencia de una arquitectura en específico, es decir, es multiplataforma e indistintamente se puede utilizar el mismo archivo en sistemas de 32 y 64 bits. En esencia, ofrece una serie de ventajas frente a otros motores de base de datos como MySQL, básicamente, presenta un mayor rendimiento, portabilidad y estabilidad.
    Finalmente, la elección del motor de base de PostgreSQL para el manejo de la información de grandes volúmenes y la sencillez con la que se generan las sentencias de envío y recepción de información, adicionalmente, argumentando que el manejo de la información no genera una amenaza de seguridad potencial.

Básicamente, Dajngo facilita a los desarrolladores el despliegue de aplicaciones sin la preocupación del manejo de dependencias extras como lo tienen otros frameworks.
    En resumen, se puede definir que las características del framework muestran su punto más fuerte, las mismas que son:

  • Completitud.
  • Versatilidad.
  • Seguridad.
  • Escalabilidad.
  • Mantenibilidad.
  • Portabilidad.

    2.5. Metodología, filosofía o arquitectura de software.

  • Metodología XP

    • En sí el trabajo con una metodología permite disminuir el nivel dificultad, tener una organización de la tareas, agilizar procesos y mejorar el resultado final.
    Está metodología es un conjunto de técnicas, que evidencia principios como el desarrollo incremental. la participación activa del cliente, el interés en las personas y no en los procesos como elemento principal, y aceptar el cambio y la simplicidad.
    La principal ventaja de XP es que ayuda a potenciar las relaciones personales mediante el trabajo en equipo, impulsando la comunicación y eliminación de tiempos muertos.

  • Arquitectura MTV

    • El patrón modelo-template-vista es el que por defecto se emplea con Django, en esencia, consta de tres componentes de vital importancia:

    Modelo

    Maneja todo lo relacionado con la información, esto incluye cómo acceder a esta, la validación, relación entre los datos y su comportamiento.

    Vista

    Representa el enlace entre las tablas y template en esencia, es la zona de administración de las vistas.

    Template

    Es la muestra de la información visual acompañada de una presentación adecuada por CSS y JS.
